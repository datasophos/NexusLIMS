nexusLIMS.harvesters.reservation_event module#

A representation of calendar reservations.

This module contains a class to represent calendar reservations and associated metadata harvest metadata from various calendar sources. The expectation is that submodules of this module will have a method named res_event_from_session implemented to handle fetching a ReservationEvent object from a nexusLIMS.db.session_handler.Session object.

class nexusLIMS.harvesters.reservation_event.ReservationEvent(experiment_title=None, instrument=None, last_updated=None, username=None, user_full_name=None, created_by=None, created_by_full_name=None, start_time=None, end_time=None, reservation_type=None, experiment_purpose=None, sample_details=None, sample_pid=None, sample_name=None, sample_elements=None, project_name=None, project_id=None, project_ref=None, internal_id=None, division=None, group=None, url=None)[source]#

A representation of a single calendar reservation.

The representation is independent of the type of calendar the reservation was made with. ReservationEvent is a common interface that is used by the record building code.

Any attribute can be None to indicate it was not present or no value was provided. The as_xml() method is used to serialize the information contained within a ReservationEvent into an XML representation that is compatible with the Nexus Facility Experiment schema.

Parameters:

  • experiment_title (str | None) – The title of the event

  • instrument (Instrument | None) – The instrument associated with this reservation

  • last_updated (datetime) – The time this event was last updated

  • username (str | None) – The username of the user indicated in this event

  • user_full_name (str | None) – The full name of the user for this event

  • created_by (str | None) – The username of the user that created this event

  • created_by_full_name (str | None) – The full name of the user that created this event

  • start_time (datetime | None) – The time this event was scheduled to start

  • end_time (datetime | None) – The time this event was scheduled to end

  • reservation_type (str | None) – The “type” or category of this event (such as User session, service, etc.)

  • experiment_purpose (str | None) – The user-entered purpose of this experiment

  • sample_details (Optional[List[str | None]]) – A list of the user-entered sample details for this experiment. The length of the list must match that given in sample_pid and sample_name.

  • sample_pid (Optional[List[str | None]]) – A list of sample PIDs provided by the user. The length of the list must match that given in sample_details and sample_name.

  • sample_name (Optional[List[str | None]]) – A list of user-friendly sample names (not a PID). The length of the list must match that given in sample_details and sample_pid.

  • project_name (Optional[List[str | None]]) – A list of the user-entered project names for this experiment. The length of the list must match that given in project_id and project_ref.

  • project_id (Optional[List[str | None]]) – A list of the specific project IDs within a research group/division. The length of the list must match that given in project_name and project_ref.

  • project_ref (Optional[List[str | None]]) – A list of (optional) links to this project in another database. The length of the list must match that given in project_name and project_id.

  • internal_id (str | None) – The identifier assigned to this event (if any) by the calendaring system

  • division (str | None) – An identifier of the division this experiment was performed for (i.e. the user’s division)

  • group (str | None) – An identifier of the group this experiment was performed for (i.e. the user’s group)

  • url (str | None) – A web-accessible link to a summary of this reservation
